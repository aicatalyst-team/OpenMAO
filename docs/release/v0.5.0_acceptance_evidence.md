# v0.5.0 Acceptance Evidence

This document maps the first "institutional memory that compounds" release to code and test evidence.

**Status:** accepted stable release
**Last updated:** 2026-05-30
**Runtime:** TypeScript on Node.js 22 LTS

## Summary

v0.5.0 ships the first Phase 2 slice: institutional memory that compounds. Knowledge **accumulates**
with independent, auditable corroboration (raising confidence, always under human approval) and
becomes **reusable** through deterministic, evidence-backed retrieval, surfaced in the CLI, API,
console, and the world model. The default runtime stays deterministic, offline, and credential-free.

## Semantics

- **Corroboration measures independence.** It comes from distinct actors and distinct individual
  memory entries, never the proposer. It raises confidence and can gate ratification, but never
  replaces human approval for a collective-memory write.
- **Retrieval is deterministic and dependency-free** (ASCII tokenization; ranked by term overlap,
  then confidence, then corroboration count, then id). Non-Latin tokenization and a search engine are
  out of scope for this release.
- **The world-model `collective_memory` summary is a rebuildable projection** of authoritative state;
  the snapshot id reflects its full nested content.

## Upgrade note

This release adds the `Corroboration` model, a `promotion_corroborations` table, and a
`collective_memory` field on the world-model snapshot. All additions are backward-compatible; a
pre-`v0.5.0` local `.openmao` database keeps working.

## Out of scope (v0.5.0)

Transactive memory (who-knows-what), confidence decay, contradiction detection, world-model-as-
decision-input, embeddings/vector search, and an automated in-demo retrieval step (which needs a
multi-run scenario to meaningfully reuse prior collective knowledge).
